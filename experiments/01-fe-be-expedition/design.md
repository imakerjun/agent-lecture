# 실험 01: FE-BE 원정대 미션 설계

## 실험 목표

AI 에이전트가 교육 설계(미션 요구사항)를 생성하고, 루브릭 기반 평가 → 개선 반복을 통해 품질을 수렴시킬 수 있는지 검증한다.

## 가설

> 교육 목표와 기대효과를 입력으로 주고, 코치 관점 + 수강생 관점 리뷰를 반복하면, 3회 이내에 루브릭 총점 20점 이상(25점 만점)의 미션 설계서를 생성할 수 있다.

## 입력

### 교육 맥락

- **대상**: 우아한테크코스 레벨2 크루
  - 프론트엔드 크루 1명 + 백엔드 크루 1명 = 페어
- **시기**: 레벨2 원정대 기간
- **전제**: 레벨1에서 JavaScript 기초, 페어프로그래밍, 단위 테스트를 학습 완료

### 미션 개요

프론트엔드 1명 + 백엔드 1명이 페어로 **타인이 사용할 수 있는 웹앱**을 만들어 배포한다.
원정대 기간 중 초기 버전을 배포하고, 레벨2 남은 기간 동안 유지보수/개선한다.

### 기대효과

1. "프론트만 하는 사람"이 아닌, 웹 애플리케이션 전체를 이해하는 개발자로 성장
2. 백엔드 크루와의 협업에서 서버 구조를 이해하고 대화할 수 있는 역량 확보
3. TDD 습관이 레벨3 이후 프로젝트에서 코드 품질을 지키는 기반이 됨
4. 레벨1 원정대에서 만든 "송곳"을 레벨2 미션에 적용해보는 연결고리
5. 프론트만으로 경험하기 어려운 TDD, 객체지향에 대해 더 깊이 고민

### 실행 단계

| 단계 | 에이전트 | 산출물 | 종료 조건 |
|------|---------|--------|-----------|
| 1. 초안 생성 | Mission Designer | `runs/run-01.md` | 미션 설계서 7개 섹션 완성 |
| 2. 평가 | Coach Reviewer + Crew Perspective | run 파일 하단에 추가 | 5개 차원 점수 + 개선 포인트 |
| 3. 개선 | Mission Designer (리뷰 반영) | `runs/run-02.md` | 이전 개선 포인트 반영 확인 |
| 4. 재평가 | Coach Reviewer + Crew Perspective | run 파일 하단에 추가 | 루브릭 20점 이상 or 3회 도달 |
| 5. 최종 리뷰 | 사람 (코치 본인) | `evaluation.md` | 실제 사용 가능 여부 판단 |

## 평가 기준

- 루브릭: [rubrics/expedition-mission-design.md](../../rubrics/expedition-mission-design.md)
- 종합 목표: **20점 이상 / 25점 만점**
- 최대 반복 횟수: **3회**

## 성공 판정

이 실험이 "성공"이란:

1. **정량**: 3회 이내에 루브릭 20점 이상 달성
2. **정성**: 코치(본인)가 "이 설계서를 기반으로 실제 미션을 운영할 수 있겠다"고 판단
3. **프로세스**: 반복 회차마다 점수가 단조증가 (개선이 실제로 품질을 올리는지)

## 실패 시 배울 점

- 루브릭 자체가 부적절한 것인지, 에이전트 프롬프트가 부족한 것인지 구분
- 사람 개입이 필요한 지점 식별 (에이전트만으로 안 되는 부분)
- 다음 실험을 위한 루브릭/프롬프트 개선 방향 도출
